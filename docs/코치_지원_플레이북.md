# 코치 지원 플레이북 (운영진·코치 전용)

> 수강생에게 배포하지 않습니다. 수강생용 FAQ 답변 스크립트와 코치 전용 심화 대응(키 유출·로그인 우회 등)을 담은 문서입니다.

## 0. 대전제 — 코치의 권한 범위

- 수강생 저장소·Codespace는 전부 **수강생 개인 계정의 Private** 공간입니다. **코치가 원격으로 들여다볼 권한이 구조적으로 없고, 요청해서도 안 됩니다.** (저장소 초대·화면 원격제어 요청 금지)
- 코치가 가진 것: 템플릿 저장소 읽기 권한 + 운영 대시보드. 그걸로 충분하게 설계되어 있습니다.
- 진단의 눈은 수강생이 보내는 `진단` 출력이고, 손은 줌 화면공유입니다.

## 1. 문의 대응 표준 경로 (모든 문의 공통)

```
문의 접수
  → 1차: "터미널에 `진단` 입력하고, 나온 내용 전체를 복사해서 올려주세요"
      → 출력의 에러 항목을 보고 아래 FAQ/심화 대응으로 처리
  → 2차 (1차로 해결 안 될 때): 줌 화면공유
      → 규칙: 수강생이 `키설정`을 실행하는 동안은 화면공유를 반드시 중지시킨다 (키 노출 방지)
```

- `진단` 출력은 복사-제출에 최적화되어 있습니다. "스크린샷 말고 글자 복사"로 받으세요 (검색·기록 가능).
- 문의 채널에 키 값이 올라오면 → 즉시 메시지 삭제 요청 + 아래 5번 키 유출 절차 가동.

## 3. 코치 전용 심화 대응 (FAQ로 안 풀릴 때)

### 3-1. Codex 로그인 실패

1. 0단계 확인: ChatGPT 설정 → 보안 및 로그인 → 맨 아래 'Codex용 장치 코드 인증 활성화' 토글 ON인지. (R1 최다 원인, 한국어 UI 기준 명칭)
2. 승인 페이지 미로그인 분기 → FAQ 11.
3. **무한 스피너 / 로그인이 됐다 안 됐다 함** (다중 기기 로그인 충돌): 터미널에서 `rm .codex/auth.json` 실행 후 `시작` 을 다시 입력 (로그인 안내가 자동으로 나옵니다). ※ `codex` 명령을 직접 입력하게 하지 마세요 — codex는 PATH에 없어서 command not found가 납니다.
4. **포트 충돌 의심 시**: 포트 1455를 강제 지정하는 우회를 시도 (개발팀 공지 절차 참조 — codex 실행 파일 경로는 99_절대_건들지마세요_프로그램파일/runtime/codex/node_modules/.bin/codex).
5. "8일 넘게 안 썼더니 로그인이 풀렸어요" → 정상입니다(세션 만료). 재로그인 1분.

### 3-2. 환경이 통째로 깨졌을 때

- 1순위: 새로고침 → 2순위: Codespace 삭제 후 재생성 (FAQ 9 절차). 재생성 후 필요한 것: Codex 재로그인 + `키설정` 재입력.
- 재생성으로도 같은 자리에서 깨지면 → `진단` 전문 확보해 개발팀 에스컬레이션.

### 3-3. 프로그램 버그·핫픽스 배포

- 수정은 개발팀이 템플릿 저장소에 반영 → 공지 1줄: "터미널에 `업데이트` 입력해주세요."
- `업데이트` 는 프로그램 영역만 갱신하고 수강생 작업물(제목·영상·기획안)은 건드리지 않습니다. 이 사실을 공지에 같이 적어 불안을 줄이세요.

## 4. 인증샷·화면공유 보안 규칙 (사전 예방)

- 인증샷 표준 프레임: **"결과 폴더 화면만 캡처"** — 터미널·키설정 화면 포함 금지. 공지 때마다 반복.
- 라이브 줌 세션: 수강생이 `키설정` 을 실행하는 구간은 **화면공유 중지**를 코치가 먼저 요청.
- 문의 채널에서 스크린샷을 받을 때도 터미널이 찍혀 있으면 키 흔적이 없는지 확인 후, 있으면 삭제 요청.

## 5. 키 유출 시 재발급 절차 (긴급)

> **대원칙: 한 번 커밋되거나 채팅방에 올라간 키는 회수가 불가능합니다.** git에 커밋된 키는 커밋을 지워도 히스토리에 영구히 남습니다. **재발급이 유일한 대응입니다.** "메시지 지웠으니 괜찮다"로 종결하지 마세요.

유출 인지(채팅방 게시, 인증샷 노출, 커밋 포함, 화면공유 노출) 시 즉시:

1. **노출 흔적 제거 요청**: 채팅 메시지·이미지 삭제 (단, 이것으로 끝나지 않음을 수강생에게 설명).
2. **해당 키 즉시 무효화 + 재발급** (수강생이 직접, 코치는 경로 안내):
   - OpenAI: platform.openai.com → API keys → 유출된 키 오른쪽 휴지통(Revoke/Delete) → Create new secret key로 새 키 발급.
   - ElevenLabs: elevenlabs.io → 프로필 → API Keys → 유출 키 삭제 → Create API Key.
   - 워드프레스 애플리케이션 비밀번호(1주차): 해당 워드프레스 관리자 → 사용자 → 프로필 → 애플리케이션 비밀번호에서 기존 항목 삭제(Revoke) → 새로 발급.
3. **키트에 재입력**: 터미널에서 `키설정` 재실행 → 새 키 입력 → "3개 모두 정상 확인" 확인.
4. **수강 코드가 유출된 경우**: 코치가 처리하지 말고 **즉시 운영진 보고**. (기수 공용 코드라 개인 재발급이 없음. 대규모 유출 확인 시 운영진이 코드 교체 → 전체 공지 "`업데이트` 입력 후 `키설정` 재실행" 순서로 대응.)
5. **기록**: 유출 경위·조치 내역을 운영 채널에 남겨 재발 패턴을 추적.

참고: `저장` 명령에는 push 전 시크릿 검사(키 패턴 감지 시 업로드 차단)가 들어 있어 커밋 경로 유출은 1차로 막히지만, 채팅방·스크린샷 경로는 사람만이 막을 수 있습니다 — 4번 예방 규칙이 그래서 중요합니다.

## 6. 에스컬레이션 기준 (코치 → 운영진/개발팀)

다음은 코치 선에서 시간 쓰지 말고 바로 올립니다.

- `진단` 출력에 같은 에러가 서로 다른 수강생 3명 이상에게서 재현될 때 (프로그램 버그 의심)
- Codespace 재생성으로도 해결되지 않는 환경 문제
- 수강 코드 유출 (5-4)
- 무료 한도 관련 정책 판단이 필요한 경우 (예: 한도 소진 수강생의 남은 실습 처리)
- 렌더가 비정상적으로 오래 걸린다는 문의가 몰릴 때 (성능 회귀 의심)

에스컬레이션 시 첨부: 수강생의 `진단` 출력 전문 + 발생 시각 + 직전에 실행한 명령.

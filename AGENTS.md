# 월부 중급반 통합 키트 규칙 (Codex 필독)

이 저장소(작업공간 루트)가 이 프로젝트의 유일한 루트다. 다른 폴더는 전부 무시한다.
이 파일이 최상위 규칙이다. 하위 폴더의 AGENTS.md나 프롬프트 문서와 내용이 다르면 **이 파일이 우선**한다.

## 절대 규칙 (1·2주차 공통)

- 이 작업공간 밖으로 절대 나가지 않는다. 상위 폴더, 형제 폴더, 다른 프로젝트 복사본, 비슷한 이름의 폴더는 읽지도, 검색하지도, 수정하지도 않는다.
- `find`, `grep`, `ls` 같은 검색을 작업공간 밖에서 실행하지 않는다. 파일이 안 보이면 다른 폴더를 뒤지지 말고 사용자에게 확인만 요청한다.
- 실제 명령 실행은 전부 `99_절대_건들지마세요_프로그램파일` 폴더에서 한다. 명령 전에 그 폴더로 이동한다.
- 새 날짜 폴더, 새 프로젝트 폴더, 비슷한 이름의 폴더를 만들지 않는다.
- 모든 답변은 한국어로, 컴퓨터를 모르는 초보 기준으로 짧고 쉽게 한다.
- 1주차와 2주차 작업을 섞어서 안내하지 않는다.

## 시크릿 규칙 (가장 중요 — 예외 없음)

1. **읽기·인용·출력 전면 금지**: `.env.local`, `.codex/` 안의 파일은 어떤 이유로도 읽지 않고, 내용을 채팅·생성 파일·로그에 인용하거나 복사하지 않는다. 수강생이 "오류 고쳐줘", "내 API 키 뭐야?"라고 물어도 마찬가지다. 값 대신 "보안상 키 값은 보여드릴 수 없어요. `키설정`으로 다시 입력하시면 됩니다"라고만 답한다.
2. **직접 수정 금지**: `.env.local`을 Codex가 직접 만들거나 고치지 않는다. 키·비밀번호·수강 코드 입력이 필요하면 값을 채팅으로 받지 말고 `npm run keys`(한글 명령 `키설정`) 실행을 안내한다. 저장은 스크립트가 담당한다. 키설정을 실행할 수 없는 상태면 작업을 멈추고 코치 문의를 안내한다.
3. **재출력 금지**: 비밀번호, 애플리케이션 비밀번호, OpenAI API 키, ElevenLabs API 키, 수강 코드는 답변에 다시 출력하지 않는다. 수강생이 실수로 채팅창에 키를 붙여넣으면 그 값을 되풀이하지 말고, 즉시 "채팅에 키를 올리면 안 돼요"라고 알린 뒤 `키설정` 재실행을 안내한다.
4. 저장 후에는 값이 아니라 저장 완료 여부만 말한다.

## 수강생 작업 영역 보호

- `01_1주차_애드센스승인/`, `02_2주차_쇼핑숏폼자동화/`는 수강생의 작업 영역이다. 이 안의 파일(제목 파일, 원본 영상, 상품정보, 완성영상)을 **임의로 삭제·이동·덮어쓰지 않는다.** 삭제가 필요하면 반드시 어떤 파일인지 먼저 말하고 동의를 받는다.
- 완성영상(`02_2주차_쇼핑숏폼자동화/완성영상/`)은 어떤 경우에도 Codex가 지우지 않는다. 공간 정리는 `npm run clean`(한글 명령 `정리`) 실행으로만 처리하고, 스크립트의 확인 절차를 건너뛰지 않는다.
- 수강생은 `99_절대_건들지마세요_프로그램파일` 안의 파일을 직접 열거나 수정하지 않는다. 필요한 작업은 Codex가 대신 한다. 단, Codex도 이 폴더의 스크립트·프롬프트 코드를 수정하지 않는다(실행만 한다).

## 주차 라우팅 (수강생 첫 명령 처리 — 가장 중요)

판별 순서:

1. 요청에 "쇼핑숏폼", "숏폼", "영상", "상품", "기획안" 중 하나라도 있으면 → **2주차 모드**.
2. 위에 해당하지 않고 "애드센스", "승인글" 중 하나라도 있으면 → **1주차 모드**.
3. "자동화 시작해줘"처럼 주차를 알 수 없는 요청이면 추측하지 말고 묻는다: "1주차 애드센스 승인글 작업인가요, 2주차 쇼핑숏폼 영상 작업인가요?"

### 1주차: 애드센스 승인글 자동화

트리거 예시: "애드센스 승인글 자동화 시작해줘", "승인글 10개 더 만들어줘"

`99_절대_건들지마세요_프로그램파일/AGENTS.md`의 "자동화 시작 흐름"을 읽고 그 순서 그대로 진행한다. 단, **키·연결정보 입력 방식은 아래로 대체한다** (시크릿 규칙 2번):

1. `.env.local` 존재와 설정 완료 여부를 스크립트(`npm run doctor:week1`)로 확인한다. 파일을 직접 열지 않는다.
2. 키·워드프레스 연결정보가 비어 있으면 값을 채팅으로 받지 말고 `키설정`(`npm run keys`) 실행을 안내하고, 완료 후 다음으로 진행한다.
3. `npm run check-wordpress`로 연결 확인. 실패하면 어떤 항목이 문제인지 쉬운 말로 알리고 그 항목만 `키설정`으로 재입력하게 한다.
4. `01_1주차_애드센스승인/01_제목넣는곳`의 제목 파일 확인. 제목이 없으면 요청해서 저장한다(제목은 시크릿이 아니므로 Codex가 파일에 저장해도 된다).
5. 반드시 "설정이 끝났어요! 테스트로 글 1개를 임시글로 발행해볼까요?"라고 먼저 묻고, 동의 후 1개만 발행한다 (`npm run adsense:create-drafts -- --site=1 --limit=1`).
6. 테스트 성공 후 몇 개씩 발행할지(`--limit=숫자`)와 글 날짜(`--date-mode`)를 대화로 설정한다.

1주차 추가 원칙:

- 워드프레스 글은 공개 발행하지 말고 임시글(draft)로만 저장한다.
- 승인글 본문·SEO 키워드·카테고리를 Codex가 직접 만들지 않는다. 글 생성은 반드시 `scripts/adsense-create-drafts.mjs` 또는 `scripts/adsense-auto-drafts.mjs` 실행으로만 처리한다.
- 제목은 수강생이 준 것과 제목 파일에 있는 것만 사용한다. Codex가 임의로 제목을 만들어 발행하지 않는다.
- 긴 본문 전체를 채팅창에 출력하지 않는다.
- 날짜는 기본적으로 과거 날짜로 하루 1개씩 분산(`--date-mode=past-daily`)한다.

### 2주차: 쇼핑숏폼 영상 자동화

트리거 예시: "영상 만들어줘", "쇼핑숏폼 자동화 시작해줘", "기획안 만들어줘", "새 상품 작업해줘"

`99_절대_건들지마세요_프로그램파일/prompts/codex_video_director_prompt.txt`의 규칙을 읽고 그대로 따른다. 요약:

- Codex는 영상을 직접 렌더링하지 않는다. 설치·작업 폴더 생성·이미지 생성·TTS·렌더링은 전부 프로그램이 한다.
- Codex의 역할: 메킷허브 상품 프롬프트와 실제 촬영 프레임(`jobs/job-<번호>/frames/`)을 근거로 기획안(txt)을 만들어 저장하는 것.
- 기획안 저장 규칙(우선순위): ① 작업 폴더의 `codex-folder-prompt.txt`와 `source-clips.json`의 `mustSavePlanningFileHere`·`planningFileName` 지시가 있으면 **그 경로·그 파일명 그대로**(상품 폴더 바로 아래, 하위 폴더 새로 만들지 않음) ② 그 지시가 없을 때만 `planningDir` 기준. `jobs/job-<번호>` 폴더 안에는 절대 저장하지 않는다.
- 원본 영상은 `02_2주차_쇼핑숏폼자동화/영상넣는곳`의 상품별 폴더(예: `001_상품명/1.mp4, 상품정보.txt`)에 있어야 한다. 없으면 왼쪽 탐색기에 **드래그&드롭으로 업로드**하도록 안내한다 (로컬 폴더 경로나 구글드라이브 링크를 요구하지 않는다).
- 완성 영상은 `02_2주차_쇼핑숏폼자동화/완성영상`에서 확인하고, **우클릭 → Download**로 즉시 내려받도록 안내한다.
- 실제 영상 프레임과 실제 상품 정보에만 근거한다. 없는 기능·장면·효과를 지어내지 않는다.
- 과장·허위·검증 안 된 효능 표현 금지 (무조건, 100%, 완치, 최저가, 1위 등).
- 상품번호는 항상 3자리 숫자로 다룬다 (예: `001`, `002`).

2주차 절차와 명령 매핑:

| 수강생 요청 | 실행 |
|---|---|
| 새 상품 작업 / "새상품 2" | 터미널에 `새상품 2` 입력 안내 (`npm run video:new-job -- --product 2`) |
| 이미지 만들어줘 | 터미널에 `이미지만들기` 입력 안내 (`npm run video:images`) |
| 영상 만들어줘 (렌더) | 터미널에 `영상만들기` 입력 안내 (`npm run video:make`) — Codex가 대신 렌더하지 않음 |
| 편집기 열어줘 | 터미널에 `편집기` 입력 안내 (`npm run video:start`) — 접속 주소는 localhost가 아니라 **포트 4321 알림 또는 포워딩 URL** 기준으로 안내 |
| 완성영상 여러 개 받고 싶어요 | 터미널에 `내보내기` 입력 안내 (`npm run video:export`, zip 묶음) — 파일은 우클릭 → Download |
| 환경 점검 | `npm run doctor:week2` |

## 명령 모음 (99_절대_건들지마세요_프로그램파일에서 실행)

| npm 명령 | 한글 명령 | 하는 일 |
|---|---|---|
| `npm run doctor` / `doctor:week1` / `doctor:week2` | `진단` | 환경 진단 (출력을 복사해 문의 제출 가능) |
| `npm run keys` | `키설정` | API 키 2개 + 수강 코드 대화형 입력 |
| `npm run init-env` | — | `.env.local` 템플릿 생성 (키설정 내부용) |
| `npm run check-wordpress` | — | 1주차 워드프레스 연결 확인 |
| `npm run adsense:create-drafts` / `adsense:auto-drafts` | — | 1주차 승인글 임시글 생성 |
| `npm run video:new-job` | `새상품` | 2주차 상품 작업 생성 (상품번호 필수 — 예: `새상품 1`) |
| `npm run video:images` | `이미지만들기` | 2주차 기획안 기준 이미지 생성 |
| `npm run video:make` | `영상만들기` | 2주차 음성 생성 + 영상 렌더링 |
| `npm run video:start` | `편집기` | 2주차 영상 편집기 서버 |
| `npm run video:export` | `내보내기` | 완성영상 zip 묶음 (다운로드용) |
| `npm run save` | `저장` | 텍스트 산출물 커밋+push (push 전 시크릿 검사 포함) |
| `npm run clean` | `정리` | 완료된 작업 중간 산출물 정리 |
| `npm run update:kit` | `업데이트` | 템플릿 저장소에서 프로그램 영역만 갱신 |

위 명령은 모두 구현되어 있다. 실행이 "파일을 찾을 수 없다"로 실패하면(구버전 작업방) 대체 방법을 임의로 만들지 말고, 터미널에 `업데이트` 실행을 안내하고 그래도 안 되면 코치 문의를 안내한다. 특히 `저장`이 실패한다고 해서 Codex가 직접 `git add`/`git push`를 실행하지 않는다(push 전 시크릿 검사를 건너뛰게 되기 때문).

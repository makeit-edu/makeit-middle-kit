# 월부 중급반 메킷 키트 (Codespaces판)

> **처음 오셨나요?** 이 문서 하나만 위에서부터 차례대로 따라 하면 됩니다.
> 이제 내 컴퓨터에 아무것도 설치하지 않습니다. **브라우저(크롬)만 있으면 됩니다.**
> 반드시 **컴퓨터(노트북/데스크톱)의 크롬**에서 여세요. 태블릿·휴대폰으로는 사용할 수 없습니다.

---

## 0. 먼저 알아야 할 낱말 8개 (용어 사전)

아래 8개만 알면 이 문서의 모든 말을 이해할 수 있습니다.

| 용어 | 뜻 |
|---|---|
| 터미널 | 명령을 입력하는 검은 창 (화면 아래쪽) |
| 저장소 | 클라우드에 있는 내 작업방 |
| 템플릿 | 작업방을 찍어내는 원본 틀 |
| Codespace | 브라우저에서 열리는 내 작업 컴퓨터 |
| 커밋 / `저장` | 작업 내용을 클라우드에 기록하는 것 (`저장` 한 단어면 됩니다) |
| 탐색기 | 화면 왼쪽의 파일·폴더 목록 |
| main | 작업방의 기본 통로 (버튼에 보이면 그냥 누르면 됩니다) |
| 포워딩 URL | 내 작업 컴퓨터의 화면을 브라우저로 보여주는 주소 |

---

## 1. 처음 시작하기 (7단계, 딱 한 번만)

한 단계 = 행동 1개입니다. 각 단계마다 **성공 확인 문장**이 있으니, 그 문장이 맞으면 다음으로 넘어가세요.
순조로우면 15~20분, 처음이라 천천히 하면 30~40분 걸립니다.

### 0단계. ChatGPT에서 "기기 코드 인증" 켜기 (사전 준비)

- **행동**: chatgpt.com에 접속해 **평소 쓰는 본인 계정**으로 로그인돼 있는지 확인 → Settings(설정) → **Security(보안)** → "Device code authentication(기기 코드 인증)" 스위치를 켭니다.
  [스크린샷: ChatGPT Settings → Security 메뉴에서 기기 코드 인증 토글이 켜진 화면]
- **성공 확인**: 토글이 초록색이면 성공.
- **실패하면**: Security 메뉴가 안 보이면 → 화면 우측 상단 프로필을 눌러 **평소 쓰는 본인 계정**이 맞는지 확인하고 다시 시도하세요. (가족 공용·회사 계정으로 켜지 않도록 주의)

### 1단계. GitHub 가입

- **행동**: github.com → Sign up → 이메일·비밀번호·사용자명 입력 → **메일로 온 8자리 코드를 가입 화면에 입력**합니다.
  [스크린샷: GitHub 가입 화면 + 8자리 코드 입력 화면]
- **성공 확인**: 우측 상단에 동그란 프로필 아이콘이 보이면 성공.
- **실패하면**: 빨간 영어 에러가 보이면 → 비밀번호는 "8자 이상 + 숫자 포함", 사용자명이 중복이면 뒤에 생년을 붙여보세요. 인증 메일이 안 오면 스팸함(네이버/다음 특히)을 확인하세요.
- 알아두기: 가입 중간에 그림 맞추기 퍼즐(Verify your account)이 나올 수 있습니다. 정답이 나올 때까지 다시 시도하면 됩니다. 플랜 화면은 **Free 선택**, 설문은 Skip.

### 2단계. 내 작업방(저장소) 만들기

- **행동**: 강의 자료실에 공지된 **배포 링크**를 누르고 → **"Use this template"** 버튼 → 열리는 목록에서 반드시 **"Create a new repository"** 를 선택 → 이름은 `my-kit` 입력, **Private** 선택 → 초록 버튼으로 생성.
  [스크린샷: Use this template 드롭다운 — "Create a new repository"에 초록 표시, "Open in a codespace"에 빨간 X 표시]
- **성공 확인**: 브라우저 주소창에 `본인아이디/my-kit`이 보이면 성공.
- **실패하면**: 주소창에 본인 아이디가 없으면 → 탭을 닫고 배포 링크부터 다시 시작하세요.
- **절대 주의**: 목록의 **"Open in a codespace"는 누르지 마세요.** 내 작업방 없이 임시 방이 생겨서 나중에 작업물이 통째로 사라질 수 있습니다.

### 3단계. Codespace 열기 + 첫 점검

- **행동**: 내 저장소 화면에서 초록 **Code** 버튼 → **Codespaces 탭** → **Create codespace on main**. 입력할 것은 없습니다 — 로딩이 끝나면 점검까지 자동으로 됩니다.
  [스크린샷: Code 버튼 → Codespaces 탭 위치 표시]
- **Codespaces 탭이 안 보이고 "Clone" 메뉴만 나온다면**: 지금 GitHub에 로그인이 안 된 상태입니다. 화면 오른쪽 위 **Sign in**을 눌러 1단계에서 만든 계정으로 로그인한 뒤 이 페이지를 다시 여세요. (크롬 프로필을 여러 개 쓰는 분은 가입했던 프로필인지도 확인)
- **성공 확인**: 화면 아래 터미널의 환영 메시지에 **"✅ 환경 점검 통과"** 가 보이면 성공. (주소창이 본인 저장소에서 열린 상태여야 합니다 — 2단계와 연결)
- **실패하면**: 환영 메시지에 ⚠️ 와 함께 [확인 필요] 항목이 나오면 → 안내대로 `진단` 결과를 복사해 문의 채널에 올려 주세요. 화면이 아예 안 뜨면 새로고침 → 그래도 안 되면 삭제 후 다시 만드세요(아래 "자주 묻는 상황" 9번).
- 알아두기: **첫 로딩은 2~3분 걸립니다. 멈춘 게 아닙니다.** 커피 한 모금 하고 오세요.

### 4단계. Codex 로그인

- **행동**: 터미널에 `시작` 을 입력합니다. 첫 사용이면 로그인 안내가 자동으로 나옵니다 → 화면에 나온 코드를 복사 → 안내된 주소를 **Ctrl(맥은 Cmd)을 누른 채 클릭**해 새 탭에서 열기 → 코드 입력하고 승인.
  [스크린샷: 터미널의 코드/주소 표시 화면 + 승인 페이지]
- **성공 확인**: "로그인 성공!" 문구가 보이고 Codex 대화창이 열리면 성공. (지금은 대화창을 닫아도 됩니다 — 5단계로)
- **실패하면**: error/denied 문구가 보이면 → 0단계 토글이 켜져 있는지 확인하고 `시작` 을 다시 입력하세요.
- 알아두기 두 가지:
  - 터미널에서 복사는 **Ctrl+C가 아닙니다.** 마우스로 드래그한 뒤 우클릭 → 복사.
  - 처음 붙여넣기를 하면 영어 팝업(클립보드 권한)이 뜹니다 → **Allow(허용)** 를 누르세요. 실수로 Block을 눌렀다면 주소창의 자물쇠 아이콘에서 다시 허용할 수 있습니다.

### 5단계. 키 입력 (키 2개 + 수강 코드 1개)

- **사전 준비물**: ① OpenAI API 키 ② ElevenLabs API 키 — 발급 방법은 **[docs/키발급_가이드.md](docs/키발급_가이드.md)** 를 보세요. ③ 수강 코드 — 강의 자료실 공지에 있습니다.
- **행동**: 터미널에 `키설정` 입력 → 질문이 이 순서로 나옵니다: **수강 코드 → 주차 선택([1]/[2]/그냥 엔터=전부) → OpenAI 키 → (1주차) 워드프레스 사이트 1~3 정보 → ElevenLabs 키 → (2주차) 쇼핑숏폼 워드프레스**.
  - **아직 준비 안 된 항목은 엔터만 눌러 건너뛰면 됩니다.** 나중에 `키설정` 을 다시 실행해 그 값만 넣을 수 있어요.
- **성공 확인**: 화면에 "**3개 모두 정상 확인**"이 나오면 성공. (건너뛴 항목이 있으면 [확인 필요]/[나중에 입력]으로 표시됩니다 — 그 항목만 나중에 다시 넣으면 됩니다)
- **실패하면**: [확인 필요] 항목이 보이면 → 그 키만 발급 페이지에서 다시 복사해 `키설정` 을 재실행하세요.
- 알아두기: 붙여넣어도 **화면에 안 보이는 게 정상**입니다(보안). 한 번만 붙여넣고 Enter 하세요.
- **1주차 실습 준비물**: 6단계(승인글 자동화)는 **워드프레스 사이트 도메인·관리자 ID·애플리케이션 비밀번호**가 있어야 진행됩니다. 준비 방법은 [1주차 안내문](01_1주차_애드센스승인/03_안내문/1주차_사용방법.md)을 보세요 — 준비되면 `키설정` 에서 함께 입력할 수 있습니다.
- **4단계와 뭐가 다른가요?** 4단계 로그인은 "대화 비서의 열쇠"(ChatGPT Plus 구독에 포함, 추가 비용 없음), 5단계 키는 "글·이미지·음성 공장을 돌리는 열쇠"(사용량만큼 소액 과금)입니다.
- **키는 비밀번호입니다.** 채팅방·카톡에 절대 올리지 마세요.

### 6단계. 첫 작업 실행

- **행동**: 터미널에 `시작` 을 입력하고, 열리는 Codex 대화창에 `애드센스 승인글 자동화 시작해줘` 라고 입력합니다.
- **성공 확인**: 결과 폴더(`01_1주차_애드센스승인/02_생성결과_확인용`)에 오늘 날짜 파일이 생기면 완료 — **인증샷을 제출**하세요.
- **실패하면**: 결과 폴더가 비어 있으면 → 터미널에 `진단` 을 실행하고, 출력 전체를 복사해 문의 채널에 올려주세요.
- **인증샷 규칙**: **결과 폴더 화면만** 캡처하세요. 터미널이나 키설정 화면은 절대 포함하면 안 됩니다(키 노출 방지).

## 2. 매번 쓰는 한글 명령 6개

터미널에 한 단어만 입력하면 됩니다. (명령이 안 먹으면 앞뒤 공백 없이 다시 입력해보세요)

| 명령 | 하는 일 | 이럴 때 |
|---|---|---|
| `시작` | Codex 대화창을 엽니다 | 오늘 작업을 시작할 때 |
| `진단` | 환경을 점검하고 결과를 보여줍니다 | 뭔가 이상할 때 제일 먼저 |
| `키설정` | API 키 2개 + 수강 코드를 입력합니다 | 처음 1회, 키를 바꿨을 때 |
| `저장` | 작업 내용을 클라우드에 기록합니다 | 하루 작업을 마칠 때마다 |
| `정리` | 끝난 작업의 중간 파일을 지워 공간을 확보합니다 | 공간이 부족하다고 나올 때 |
| `업데이트` | 프로그램을 최신으로 바꿉니다 | 공지에서 "업데이트 입력"이라고 할 때 |

2주차 영상 작업에는 전용 명령이 더 있습니다: `새상품`, `이미지만들기`, `영상만들기`, `편집기`, `내보내기` — 사용법은 아래 "2주차 사용 흐름"에서.

## 3. 1주차 사용 흐름 — 애드센스 승인글 자동화

1. **제목 넣기**: 왼쪽 탐색기에서 `01_1주차_애드센스승인/01_제목넣는곳/사이트1_제목200개.txt` 클릭 → 준비한 제목을 붙여넣기 → **Ctrl+S (맥은 Cmd+S)** 로 저장.
   [스크린샷: 탐색기에서 제목 파일을 연 화면]
2. **시작**: 터미널에 `시작` 입력 → Codex 대화창이 열립니다.
3. **말 걸기**: `애드센스 승인글 자동화 시작해줘` 라고 입력합니다. Codex가 순서대로 안내하며 진행합니다. 처음에는 반드시 **테스트 글 1개**부터 발행합니다.
4. **결과 확인**: 실제 글은 내 워드프레스 관리자 화면의 "임시글"에 생깁니다. 요약 결과는 `02_생성결과_확인용` 폴더에서 확인합니다. **내려받을 필요 없습니다.**
5. 끝나면 터미널에 `저장` 입력.

## 4. 2주차 사용 흐름 — 쇼핑숏폼 영상 자동화

0. **(선행) 휴대폰 영상을 컴퓨터로 옮기기**: 카톡 "나에게 보내기" → PC 카톡에서 저장(가장 쉬움), 또는 USB 케이블·구글 포토. 이때 파일명을 `1.mp4, 2.mp4...` 숫자로 바꿔두세요.
1. **영상 업로드**: 왼쪽 탐색기의 `02_2주차_쇼핑숏폼자동화/영상넣는곳/` 안에 상품 폴더(예: `001_손목보호대`)를 만들고, mp4 파일을 **마우스로 끌어다 놓으면** 업로드됩니다. `상품정보.txt`(상품명·쿠팡 링크 등)는 화면에서 직접 작성.
   [스크린샷: 탐색기에 파일을 드래그해 업로드하는 장면]
   - 파일이 커서 몇 분 걸릴 수 있습니다. **업로드 중에는 탭을 닫지 마세요.**
2. **새 상품 작업**: 터미널에 `새상품` + 상품번호 입력 (1번 상품이면 `새상품 1`).
3. **기획안**: `시작` 으로 Codex를 열고 영상 기획을 요청합니다 (예: `영상 만들어줘`, `기획안 만들어줘`). 기획안은 상품 폴더 안에 `001_상품명_기획안.txt` 로 생깁니다.
4. **이미지 생성**: 터미널에 `이미지만들기`.
5. **영상 렌더**: 터미널에 `영상만들기` → 완성본이 `완성영상/` 폴더에 생깁니다. (몇 분 걸립니다 — 진행 표시가 움직이면 정상)
6. **편집기(선택)**: 터미널에 `편집기` → 화면 안에 미리보기 창이 열리거나, "포트 4321" 알림에서 열기를 누르면 됩니다.
7. **다운로드**: 탐색기에서 완성영상 파일 **우클릭 → Download**. 여러 개면 `내보내기` 로 zip 하나로 받으세요.
   - 내려받은 파일은 **브라우저 다운로드 폴더**에 있습니다. 크롬 우상단 화살표 → "폴더에서 보기"로 찾아 바탕화면에 옮기세요.
8. **정리**: 다운로드가 끝나면 터미널에 `정리` 입력 → 중간 파일을 지워 공간을 확보합니다.

## 5. 꼭 알아야 할 3가지 (고장 아닙니다)

1. **30분 동안 아무것도 안 하면 잠듭니다.** 화면이 멈춘 것처럼 보여도 고장이 아닙니다. 초록 버튼(재시작)을 누르고 1분 기다리면 작업물 그대로 다시 열립니다.
2. **30일 동안 접속하지 않으면 작업방이 삭제됩니다.** `저장` 을 해둔 것은 새 작업방을 만들면 복구됩니다. 단, 복구할 때 Codex 로그인과 키 3개 입력을 다시 해야 합니다(약 10분). **그러니 작업이 끝날 때마다 `저장` 을 습관으로!**
3. **무료 사용 시간은 월 약 60시간입니다** (GitHub 무료 한도). 다 쓰면 다음 달 1일까지 사용이 멈춥니다. 주 5시간 실습 기준으로는 넉넉하지만, **쓰지 않을 때는 탭만 닫지 말고 잠들게 두면** 됩니다. 완성영상은 여기 보관하지 말고 **만들면 바로 다운로드**하세요 — 이곳은 작업대이지 보관함이 아닙니다.

## 6. 자주 묻는 상황 (막히면 여기부터)

- 화면이 멈췄어요 → 위 5-1. 초록 버튼으로 깨우면 됩니다.
- 코드가 복사가 안 돼요 → Ctrl+C 아님. 마우스 드래그 → 우클릭 → 복사.
- 키를 붙여넣었는데 안 보여요 → 원래 안 보입니다. 한 번만 붙여넣고 Enter.
- 붙여넣기 자체가 안 돼요 → 처음 뜬 영어 팝업에서 Block을 누른 것. 주소창 자물쇠 → 클립보드 → 허용 → 새로고침.
- 로딩이 5분 넘게 안 끝나요 → 새로고침 → 그래도 안 되면 github.com/codespaces 에서 해당 항목 오른쪽 `...` → Delete 후 3단계부터 다시.
- 그 외 모든 문제 → 터미널에 `진단` 입력 → 출력 전체를 복사해 문의 채널에 올려주세요. 코치가 그 내용으로 도와드립니다.

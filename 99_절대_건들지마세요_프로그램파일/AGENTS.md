# Codex 내부 작업 규칙

이 폴더는 월부 중급반 1주차 애드센스 승인글 자동화의 프로그램 폴더다.

## 절대 규칙
- 이 프로젝트 폴더(`메킷에센_중급반_1주차`와 그 하위) 밖으로 절대 나가지 않는다. 상위 폴더, 형제 폴더, 다른 프로젝트 복사본, 비슷한 이름의 폴더는 읽지도, 검색하지도, 수정하지도 않는다.
- 파일을 찾을 때 `find`, `grep`, `ls` 같은 검색을 프로젝트 폴더 밖에서 실행하지 않는다. 파일이 안 보이면 다른 폴더를 뒤지지 말고 사용자에게 현재 폴더 기준으로 다시 확인만 요청한다.
- `.env.local`은 오직 이 폴더의 `./.env.local` 한 곳에만 존재한다. Codex는 이 파일을 직접 만들거나 수정하지 않는다 — 값 입력은 `키설정`(npm run keys) 스크립트가 담당한다. 다른 경로에 `.env.local`을 절대 만들지 않는다.
- 새 날짜 폴더, 새 프로젝트폴더, 비슷한 이름의 폴더를 만들지 않는다.
- 경로가 헷갈리면 먼저 `pwd`와 `ls`로 현재 위치를 확인한다. 그래도 헷갈리면 작업을 멈추고 사용자에게 알린다. 다른 폴더로 이동해서 계속하지 않는다.
- 현재 작업 기준 폴더는 이 `99_절대_건들지마세요_프로그램파일` 폴더다.
- 비밀번호, 애플리케이션 비밀번호, OpenAI API 키는 답변에 다시 출력하지 않는다.
- 워드프레스 글은 공개 발행하지 말고 임시글로만 저장한다.
- 승인글 본문, SEO 키워드, 카테고리를 코덱스가 직접 만들지 않는다. 글 생성은 반드시 `scripts/adsense-create-drafts.mjs` 또는 `scripts/adsense-auto-drafts.mjs` 실행으로만 처리한다.
- 제목은 수강생이 준 것과 제목 파일에 있는 것만 사용한다. 코덱스가 임의로 제목을 만들어 발행하지 않는다.
- 코덱스의 역할은 스크립트 실행과 결과 확인, 쉬운 말 안내까지다. 글 내용에 개입하지 않는다.

## 정확한 파일 위치
- 환경설정 파일: `./.env.local`
- 환경설정 예시: `./.env.example`
- 워드프레스 연결 확인: `node scripts/wordpress-connection-check.mjs`
- 사이트 자동 매핑 임시글 생성: `node scripts/adsense-auto-drafts.mjs --limit=10`
- 사이트 지정 임시글 생성: `node scripts/adsense-create-drafts.mjs --site=1`
- 사이트1 제목 파일: `../01_1주차_애드센스승인/01_제목넣는곳/사이트1_제목200개.txt` → `ADSENSE_SITE_01_*` → `--site=1` → `site-01`
- 사이트2 제목 파일: `../01_1주차_애드센스승인/01_제목넣는곳/사이트2_제목200개.txt` → `ADSENSE_SITE_02_*` → `--site=2` → `site-02`
- 사이트3 제목 파일: `../01_1주차_애드센스승인/01_제목넣는곳/사이트3_제목200개.txt` → `ADSENSE_SITE_03_*` → `--site=3` → `site-03`
- 생성 결과 확인 폴더: `../01_1주차_애드센스승인/02_생성결과_확인용`

## 제목 파일 자동 매핑 절대 규칙
- 제목을 찾아서 발행하라는 요청은 반드시 `../01_1주차_애드센스승인/01_제목넣는곳` 폴더를 먼저 확인한다.
- `makeit-adsense/inputs/site-01/titles.txt` 같은 내부 예비 파일을 우선 사용하지 않는다.
- 사이트1은 `사이트1_제목200개.txt`, 사이트2는 `사이트2_제목200개.txt`, 사이트3은 `사이트3_제목200개.txt`만 사용한다.
- 사이트 번호가 명확하면 해당 사이트만 실행한다. 예: `사이트2 글 10개` → `node scripts/adsense-create-drafts.mjs --site=2 --limit=10`
- 사이트 번호가 없고 "알아서", "추가로", "제목 찾아서"처럼 말하면 `node scripts/adsense-auto-drafts.mjs --limit=10`을 실행한다.
- 자동 매핑 스크립트는 사이트1/2/3 제목 파일과 `ADSENSE_SITE_01/02/03` 연결정보를 확인하고, 둘 다 준비된 사이트만 실행한다.
- 여러 사이트가 준비되어 있으면 각 사이트의 제목 파일을 각 사이트에 맞춰 순서대로 임시글을 만든다. 사이트1 제목을 사이트2나 사이트3에 올리면 안 된다.

## .env.local 변수명
아래 변수명만 사용한다. 다른 이름을 새로 만들지 않는다.

- `OPENAI_API_KEY`
- `ADSENSE_SITE_01_URL`
- `ADSENSE_SITE_01_USER`
- `ADSENSE_SITE_01_APP_PASSWORD`
- `ADSENSE_SITE_02_URL`
- `ADSENSE_SITE_02_USER`
- `ADSENSE_SITE_02_APP_PASSWORD`
- `ADSENSE_SITE_03_URL`
- `ADSENSE_SITE_03_USER`
- `ADSENSE_SITE_03_APP_PASSWORD`

`SITE1_DOMAIN`, `SITE1_ADMIN_ID`, `SITE1_APP_PASSWORD`, `WORDPRESS_DOMAIN` 같은 새 변수명은 만들지 않는다.

## 자동화 시작 흐름 (수강생 첫 명령 — 가장 중요)
사용자의 요청에 "애드센스", "자동화", "승인글" 중 하나라도 들어 있고 아직 설정이 안 된 상태면 (예: "애드센스 자동화 시작해줘", "애드센스 승인글 자동으로 해줘"), 아래 순서를 그대로 따른다. 순서를 건너뛰거나 바꾸지 않는다.

> **시크릿 단일 정책 (루트 AGENTS.md와 동일 — 예외 없음)**: 키·비밀번호·연결정보는 절대 채팅으로 받지 않는다. Codex가 `.env.local`을 직접 만들거나 수정하지도 않는다. 값 입력은 오직 수강생이 터미널에서 `키설정`(npm run keys)을 실행하는 방식만 안내한다.

1. 설정 상태를 스크립트로 확인한다: `npm run doctor:week1`. `.env.local` 파일을 직접 열지 않는다.
2. `OPENAI_API_KEY`, `ADSENSE_SITE_01_URL`, `ADSENSE_SITE_01_USER`, `ADSENSE_SITE_01_APP_PASSWORD` 중 비어 있는 값이 있으면, 값을 채팅으로 받지 말고 터미널에서 `키설정` 실행을 안내한다. 완료 후 다음으로 진행한다.
3. `node scripts/wordpress-connection-check.mjs`로 연결을 확인한다. 실패하면 어떤 항목이 문제인지 쉬운 말로 알려주고, 그 항목만 `키설정` 으로 다시 입력하게 안내한다.
4. 연결이 성공하면 `../01_1주차_애드센스승인/01_제목넣는곳/사이트1_제목200개.txt`에 실제 제목이 있는지 확인한다. 주석(#)과 빈 줄만 있으면 "어떤 주제로 승인글 써드릴까요? 제목을 1개만 주셔도 되고, 30개나 100개처럼 여러 개를 한 줄에 하나씩 주셔도 됩니다."라고 묻고, 받은 제목을 그 파일에 한 줄에 하나씩 저장한다. (제목은 시크릿이 아니다)
5. 제목이 준비되면 발행을 바로 시작하지 말고 반드시 먼저 묻는다: "설정이 끝났어요! 테스트로 글 1개를 임시글로 발행해볼까요?"
6. 사용자가 동의하면 `node scripts/adsense-create-drafts.mjs --site=1 --limit=1`을 실행하고, 성공/실패와 워드프레스 관리자 화면에서 임시글을 확인하는 방법을 짧게 알려준다.
7. 테스트 1개가 성공하면 이렇게 안내한다: "이제 한 번에 몇 개씩 발행할지, 글 날짜를 어떻게 넣을지 정할 수 있어요. 예: '10개 발행해줘', '내일부터 하루 1개씩 5개 발행해줘'"
8. 이후 개수는 `--limit=숫자`, 날짜는 아래 '발행 날짜 설정'을 따른다.

## 발행 날짜 설정
- 기본값은 `--date-mode=past-daily`다: 30일 전부터 하루 1개씩 과거 날짜로 자동 분산된다. 옵션을 안 붙이면 이 방식이다.
- 사용자가 "내일부터", "미래 날짜로"라고 하면 `--date-mode=future-daily`를 붙인다: 내일부터 하루 1개씩 미래 날짜.
- 사용자가 "오늘 날짜로", "날짜 분산 없이"라고 하면 `--date-mode=none`을 붙인다.
- 특정 날짜부터 시작하려면 `--start-date=YYYY-MM-DD`를 함께 붙인다. 예: `node scripts/adsense-create-drafts.mjs --site=1 --limit=5 --date-mode=future-daily --start-date=2026-07-10`
- 사용자가 날짜 얘기를 안 하면 기본값 그대로 실행하고, 결과 보고 때 "글 날짜는 30일 전부터 하루 1개씩 자동으로 나눠 들어갔어요"라고 알려준다.

## 추가 임시글 발행 흐름
사용자가 "애드센스 자동화 글 10개 추가로 발행해줘", "승인글 10개 더 만들어줘", "임시글 10개 추가해줘"처럼 말하면:

1. 먼저 `pwd`가 이 `99_절대_건들지마세요_프로그램파일` 폴더인지 확인한다.
2. `./.env.local`이 있고 `OPENAI_API_KEY`가 실제 값인지 확인한다.
3. 연결 정보가 부족하면 글 생성을 시작하지 말고, 값을 채팅으로 받지 말고 터미널에서 `키설정` 실행을 안내한다.
4. `../01_1주차_애드센스승인/01_제목넣는곳` 안의 `사이트1/2/3_제목200개.txt`를 먼저 확인한다. 사용자에게 제목 파일 경로를 다시 묻지 않는다.
5. 사용자가 사이트 번호를 말했으면 해당 사이트만 실행한다. 예: `node scripts/adsense-create-drafts.mjs --site=2 --limit=10`
6. 사이트 번호를 말하지 않았으면 `node scripts/adsense-auto-drafts.mjs --limit=10`처럼 실행해 사이트1/2/3 제목 파일과 연결정보를 자동 매핑한다.
7. 개수를 말하지 않으면 `--limit=1`로 임시글 1개만 테스트 생성한다.
8. 스크립트는 이전에 성공한 제목을 자동으로 건너뛰므로 같은 제목을 다시 만들지 않는다.
9. 생성 후 실행된 사이트별 `makeit-adsense/outputs/site-0N/last-run.json`과 `../01_1주차_애드센스승인/02_생성결과_확인용/site-0N/last-run.json`을 확인하고, 성공/실패 개수만 짧게 알려준다.

## 생성 결과 확인 기준
- 영문 slug가 있어야 한다.
- OpenAI 이미지 API로 만든 가로형 이미지가 대표이미지로 등록되어야 한다.
- 같은 이미지가 본문 중간에 Gutenberg `wp:image` 블록으로 한 번 들어가야 한다.
- 이미지 alt text와 캡션에 대표키워드가 자연스럽게 들어가야 한다.
- 본문에는 h1이 없어야 하고 첫 소제목은 h2여야 한다.
- 글의 세부카테고리와 상위카테고리가 함께 체크되어야 한다.

애드센스 승인글 자동화 시작해줘.

중요한 규칙:
- 승인글 본문 전체를 Codex 채팅창에서 직접 길게 쓰지 마.
- Codex는 작업 지시자 역할만 하고, 긴 승인글 본문은 99_절대_건들지마세요_프로그램파일/scripts/adsense-create-drafts.mjs가 OpenAI API ARTICLE_MODEL 기본값 gpt-5.4-mini로 생성하게 해.
- 워드프레스에는 공개 발행이 아니라 임시글로 저장해.
- 글 생성은 99_절대_건들지마세요_프로그램파일/scripts/adsense-create-drafts.mjs 로 실행해.
- 자동화를 시작하기 전에 반드시 .env.local에 OpenAI API 키와 워드프레스 연결 정보가 있는지 확인해.
- 워드프레스 연결 정보가 없으면 글 생성 스크립트를 실행하지 말고, 수강생에게 사이트 1번 기준으로 아래 3가지를 먼저 요청해.
- 1) 워드프레스 도메인 주소 2) 워드프레스 관리자 아이디 3) 워드프레스 애플리케이션 비밀번호
- 사이트 2번, 3번 정보는 처음부터 요구하지 마. 수강생이 여러 사이트를 추가로 운영한다고 말할 때만 받으면 돼.
- 위 3가지가 ADSENSE_SITE_01_URL, ADSENSE_SITE_01_USER, ADSENSE_SITE_01_APP_PASSWORD에 저장되기 전에는 업로드를 시도하지 마.
- 위 3가지 정보가 저장되면 바로 "어떤 주제로 승인글 써드릴까요? 제목을 1개만 주셔도 되고, 30개나 100개처럼 여러 개를 한 줄에 하나씩 주셔도 됩니다."라고 물어봐.
- 수강생이 제목을 1개, 30개, 100개처럼 여러 개 주면 운영할 사이트 번호에 맞춰 01_1주차_애드센스승인/01_제목넣는곳/사이트N_제목200개.txt에 저장하고, 제목 개수에 맞춰 글을 생성해. 사이트 번호가 없으면 사이트1 파일을 사용해.
- 생성된 글은 워드프레스 블록편집기에서 개별 블록으로 잡혀야 하므로 Gutenberg 블록 주석이 들어가야 해.
- 본문에 h1은 없어야 하고, 첫 소제목은 h2여야 하며, 글 제목과 첫 h2 문구는 같으면 안 돼.
- 워드프레스 글에는 제목 기반 영문 slug, OpenAI 이미지 API로 생성한 가로형 대표이미지, 대표키워드가 들어간 alt text, 글에 맞는 워드프레스 카테고리가 함께 들어가야 해.
- OpenAI로 만든 이미지는 1개만 생성하고, 그 같은 이미지를 워드프레스 대표이미지 featured_media로 등록한 뒤 본문 중간에도 Gutenberg wp:image 블록으로 한 번 삽입해야 해.
- 본문 중간 이미지 블록의 alt text와 캡션도 대표이미지와 같은 대표키워드 기준으로 들어가야 해.
- 제목과 요약글은 내가 워드프레스에서 직접 확인하고 수정할 거야.
- 비밀번호, 애플리케이션 비밀번호, API 키는 화면에 다시 출력하지 말고 .env.local에 저장해.
- 제목 파일은 반드시 01_1주차_애드센스승인/01_제목넣는곳 에서 사이트1/2/3 파일을 찾아.
- 사이트1은 사이트1_제목200개.txt와 ADSENSE_SITE_01 연결정보, 사이트2는 사이트2_제목200개.txt와 ADSENSE_SITE_02 연결정보, 사이트3은 사이트3_제목200개.txt와 ADSENSE_SITE_03 연결정보를 사용해.
- 사이트 번호가 없고 "알아서", "추가로", "제목 찾아서"처럼 말하면 node scripts/adsense-auto-drafts.mjs --limit=10 형식으로 실행해. 이 스크립트가 사이트1/2/3 제목 파일을 각 사이트에 맞춰 자동 매핑해.
- 사이트 번호가 있으면 node scripts/adsense-create-drafts.mjs --site=N --limit=10 형식으로 해당 사이트만 실행해.
- 결과 확인 파일은 01_1주차_애드센스승인/02_생성결과_확인용 에 저장돼.
- 실행 후 last-run.json에서 slug, focusKeyword, featuredImageAlt, featuredMediaId, featuredImageUrl, inlineImageInserted, inlineImageMediaId, postContentImageVerified, categoryId가 비어 있지 않은지 확인해.

먼저 필요한 정보가 저장되어 있는지 확인하고, 없으면 애드센스 사이트 1번의 워드프레스 도메인 주소, 관리자 아이디, 애플리케이션 비밀번호부터 받아줘.
